4. PROBLEME CRITICE GĂSITE
================================================================================

🚨 CRITIC #1: TOATE testele eșuează la parsare TypeScript (Babel)
   Testele conțin sintaxă TypeScript avansată (React.ComponentProps<typeof Button>)
   pe care Babel-ul din Jest nu o poate parsa fără @babel/preset-typescript.
   Eroare: "Unexpected token, expected ','"
   IMPACT: CI-ul eșuează. 0% acoperire reală de teste.

🚨 CRITIC #2: jest.config.ts are un typo — cheie invalidă
   Linia: setupFilesAfterSetup: ["<rootDir>/jest.setup.ts"]
   Problema 1: "setupFilesAfterSetup" NU este o cheie validă în Jest.
   Cheile corecte sunt: setupFiles (nu setupFilesAfterSetup).
   Problema 2: Fișierul jest.setup.ts NU există.
   IMPACT: Configurația Jest este ignorată parțial — setup-ul de teste nu rulează.

🚨 CRITIC #3: Modulele esențiale de securitate lipsesc
   src/lib/auth/ — fără nextauth-config, RBAC, ABAC, session management.
   Middleware-ul importă jose dar nu poate valida sesiuni reale.
   Autentificarea admin este hardcodată (email/parolă în clar în cod).
   IMPACT: Securitate zero, credentiale hardcodate expuse în source code.

🚨 CRITIC #4: ~70% din paginile planificate lipsesc
   Din 50+ pagini planificate, doar 5 există.
   IMPACT: Aplicația e doar un landing page + admin de bază.

🚨 CRITIC #5: ~97% din rutele API lipsesc (1 din 31 există)
   Doar GET/PUT /api/settings funcționează.
   Toate celelalte endpoint-uri (auth, users, projects, shop, AI, payments etc.)
   NU există.
   IMPACT: Backend-ul e practic nefuncțional pentru orice altceva decât setări.

🚨 ALERTĂ #6: Credențiale admin hardcodate în cod
   src/app/(admin)/login/page.tsx conține:
   const ADMIN_EMAIL = "nexusdevstudio@admin.ro";
   const ADMIN_PASSWORD = ".salataverde123!";
   Acestea sunt vizibile în client-side bundle.
   IMPACT: Risc major de securitate. Orice vizitator poate vedea parola admin.

🚨 ALERTĂ #7: Dependențele npm nu sunt instalate
   Nu există node_modules/. Proiectul nu a rulat niciodată pnpm install.
   IMPACT: Nimic nu poate fi compilat sau testat fără instalare prealabilă.

🚨 ALERTĂ #8: Nu există niciun fișier .env sau .env.example
   Fără variabile de mediu (DATABASE_URL, NEXTAUTH_SECRET, REDIS_URL, etc.)
   aplicația nu poate funcționa cu baza de date reală.
   IMPACT: Doar setările hardcodate funcționează. Orice feature cu DB e blocată.

🚨 WARNING #9: Fișierul $null din rădăcină
   Un fișier numit "$null" poate încă exista în root (artifact de scripting).
   Trebuie verificat și șters dacă există.

🚨 WARNING #10: i18n config din next.config.js poate cauza probleme
   next.config.js folosește i18n.domains (deprecated în Next.js 14 App Router).
   Configurația i18n de la nivelul next.config.js este incompatibilă cu
   middleware-ul de locale detection — pot apărea conflicte.
   Recomandare: scoate i18n din next.config.js și folosește doar middleware + next-intl.
